# 一个简化的 AI 算法教程：用神经网络区分英文和拼音

本文以 `solution.cpp` 为例，讲解一个非常小的神经网络如何完成文本分类任务：判断一段输入更像英文，还是更像汉语拼音。

本教程对应的练习题来源于洛谷：[`B3759 [信息与未来 2021] 文本分类`](https://www.luogu.com.cn/problem/B3759)。读者可以在洛谷提交对应程序，用平台测试数据验证算法是否正确。

这个程序不是调用现成 AI 库，而是手写了神经网络最核心的部分：

- 特征提取
- 前向传播
- 反向传播训练
- 多数投票分类

## 1. 问题是什么

在洛谷原题 [`B3759 [信息与未来 2021] 文本分类`](https://www.luogu.com.cn/problem/B3759) 中，输入一段由小写字母组成的文本，程序要输出：

```text
English
```

或：

```text
Pinyin
```

例如：

```text
this problem has a simple solution
```

应该判断为英文。

```text
zhe ge ti mu qi shi hen jian dan
```

应该判断为拼音。

## 2. 整体思路

程序没有直接判断整句话，而是先判断每个单词。

流程如下：

```text
一句话
  ↓
拆成多个单词
  ↓
每个单词提取 7 个数字特征
  ↓
送入 7-4-1 神经网络
  ↓
每个单词投一票
  ↓
英文票多则输出 English，否则输出 Pinyin
```

核心代码逻辑是：

```cpp
if (nn.predict(word_feat(w)) > 0.5) eng++;
```

含义是：

- `word_feat(w)`：把单词变成数字特征
- `predict(...)`：神经网络预测这个单词像不像英文
- 输出值大于 `0.5`：认为这个词是英文
- 最后用 `eng > n / 2` 做多数投票

## 3. 什么是特征

神经网络不能直接理解字符串，比如 `quick`、`zhe`、`xiang`。

所以我们要先把单词转换成一组数字，这组数字就叫特征。

程序中每个单词会被转换成 7 维特征：

| 下标 | 特征 | 作用 |
|---|---|---|
| `f[0]` | 词长 / 8 | 拼音通常较短 |
| `f[1]` | 是否含 `q` | `q` 在拼音中很常见 |
| `f[2]` | 是否含 `x` | `x` 在拼音中很常见 |
| `f[3]` | 是否含 `zh/ch/sh` | 拼音中的典型声母组合 |
| `f[4]` | 词尾是否不像拼音 | 英文常以 `s/d/k/t` 等结尾 |
| `f[5]` | 是否有连续辅音 | 英文常有 `th/br/st/ck` 等 |
| `f[6]` | 元音占比 | 拼音的元音比例往往较高 |

例如 `quick`：

```text
quick → [0.625, 1, 0, 0, 1, 1, 0.4]
```

解释：

- 长度是 5，所以 `5 / 8 = 0.625`
- 含有 `q`
- 不含 `x`
- 不含 `zh/ch/sh`
- 以 `k` 结尾，不像拼音结尾
- 有 `ck` 连续辅音
- 5 个字母中有 2 个元音，元音占比 `2 / 5 = 0.4`

例如 `zhe`：

```text
zhe → [0.375, 0, 0, 1, 0, 0, 0.5]
```

它含有 `zh`，这是很强的拼音特征。

## 4. 神经网络结构

这个程序使用的是一个很小的神经网络：

```text
输入层 7 个节点 → 隐藏层 4 个节点 → 输出层 1 个节点
```

也就是常说的：

```text
7-4-1 神经网络
```

结构如下：

```text
f[0] ──┐
f[1] ──┤
f[2] ──┼──→ H[0] ──┐
f[3] ──┤    H[1] ──┼──→ output
f[4] ──┤    H[2] ──┤
f[5] ──┤    H[3] ──┘
f[6] ──┘
```

其中：

- 输入层：接收 7 个特征
- 隐藏层：综合分析这些特征
- 输出层：输出一个 0 到 1 之间的分数

输出含义：

```text
接近 1 → 更像英文
接近 0 → 更像拼音
```

## 5. 权重和偏置是什么

神经网络真正学习的内容是参数。

程序中有 4 组参数：

```cpp
double W1[NH][NI];
double b1[NH];
double W2[NH];
double b2;
```

含义如下：

| 参数 | 含义 |
|---|---|
| `W1` | 输入层到隐藏层的权重 |
| `b1` | 隐藏层偏置 |
| `W2` | 隐藏层到输出层的权重 |
| `b2` | 输出层偏置 |

可以把权重理解为“某个特征有多重要”。

例如：

- 如果 `has_zhcs` 对判断拼音很重要，相关权重会被训练成较大的值
- 如果某个特征帮助不大，相关权重影响就会较小

## 6. Sigmoid 函数

神经网络中使用了 `sigmoid` 函数：

```cpp
double sigmoid(double x) {
    return 1.0 / (1.0 + exp(-x));
}
```

它的作用是把任意数字压缩到 `(0, 1)` 之间。

```text
很小的数 → 接近 0
0        → 0.5
很大的数 → 接近 1
```

这样输出结果就可以当成“像英文的概率分数”。

## 7. 前向传播

前向传播就是从输入算出预测结果。

隐藏层计算：

```text
h[j] = sigmoid(b1[j] + W1[j][0]f[0] + W1[j][1]f[1] + ... + W1[j][6]f[6])
```

输出层计算：

```text
out = sigmoid(b2 + W2[0]h[0] + W2[1]h[1] + W2[2]h[2] + W2[3]h[3])
```

对应代码：

```cpp
double forward(const vector<double>& in, double hid[NH]) {
    for (int j = 0; j < NH; j++) {
        double z = b1[j];
        for (int i = 0; i < NI; i++)
            z += W1[j][i] * in[i];
        hid[j] = sigmoid(z);
    }

    double z = b2;
    for (int j = 0; j < NH; j++)
        z += W2[j] * hid[j];
    return sigmoid(z);
}
```

这一步只负责“预测”，还不会学习。

## 8. 训练数据

程序准备了一些已知答案的样本。

英文样本：

```cpp
"the", "this", "brown", "quick", "jumps",
"is", "has", "was", "read", "find",
"have", "will", "of", "in", "to"
```

拼音样本：

```cpp
"qi", "xiang", "xue", "xiao", "qing",
"zhi", "shi", "zhan", "chang", "cheng",
"zhe", "ge", "de", "he", "men"
```

英文目标值是 `1.0`，拼音目标值是 `0.0`。

```cpp
for (auto w : en) data.push_back({word_feat(w), 1.0});
for (auto w : cn) data.push_back({word_feat(w), 0.0});
```

## 9. 反向传播

反向传播就是根据预测错误来调整权重。

如果目标是英文：

```text
target = 1.0
```

但神经网络输出：

```text
out = 0.3
```

说明预测太低，需要把输出往 1 的方向调整。

如果目标是拼音：

```text
target = 0.0
```

但神经网络输出：

```text
out = 0.8
```

说明预测太高，需要把输出往 0 的方向调整。

程序中输出层误差是：

```cpp
double d_out = (target - out) * sig_d(out);
```

其中：

- `target - out` 表示预测方向错了多少
- `sig_d(out)` 是 sigmoid 的导数，表示当前还能调整多少

隐藏层误差是：

```cpp
double d_hid = d_out * W2[j] * sig_d(hid[j]);
```

含义是：输出层的错误通过 `W2[j]` 反向传回隐藏层。

## 10. 学习率 lr

训练函数中有一个参数：

```cpp
void train(const vector<double>& in, double target, double lr)
```

`lr` 是学习率，也就是每次更新参数的步长。

```text
lr 太大 → 参数变化太猛，可能震荡
lr 太小 → 学得太慢，需要很多轮
```

本程序使用：

```cpp
nn.train(p.first, p.second, 5.0);
```

虽然 `5.0` 看起来比较大，但这个任务的数据很少、网络很小，所以可以快速拟合。

## 11. 参数更新

训练时会更新 4 类参数：

```cpp
W1[j][i] += lr * d_hid * in[i];
b1[j]    += lr * d_hid;
W2[j]    += lr * d_out * hid[j];
b2       += lr * d_out;
```

直观理解：

- 如果某个特征对正确答案有帮助，就增强它的权重
- 如果某个特征导致错误，就削弱它的权重
- 重复训练很多轮后，网络逐渐学会区分英文和拼音

程序训练 200 轮：

```cpp
for (int epoch = 0; epoch < 200; epoch++)
    for (auto& p : data)
        nn.train(p.first, p.second, 5.0);
```

## 12. 为什么要多数投票

单个单词可能会判断错。

例如某些英文短词和拼音很像：

```text
in, to, he
```

某些拼音也可能看起来像英文：

```text
men, de, he
```

所以程序不靠一个词决定整句话，而是让所有词投票。

```cpp
cout << (eng > n / 2 ? "English" : "Pinyin") << '\n';
```

这样即使少数单词判断错，只要大部分单词判断正确，整句结果仍然正确。

## 13. 这个程序为什么能通过

它能通过的关键不是网络很大，而是特征选得合适。

特别重要的特征有：

- `q/x`：拼音中非常常见
- `zh/ch/sh`：拼音的典型组合
- 词尾规则：拼音结尾范围很有限
- 连续辅音：英文中更常见
- 投票机制：降低单词级误判的影响

这说明一个重要思想：

```text
小模型 + 好特征 + 合理训练，也可以解决实际问题。
```

## 14. 进一步简化

如果程序已经训练好了，其实可以把训练后的权重保存下来。

这样提交版本只需要：

- `word_feat`：提取特征
- `predict`：用固定权重预测
- `main`：输入输出和投票

这就是 `aiEnglish.cpp` 的思路：训练在开发阶段完成，运行时只做推理。

## 15. 总结

这个例子展示了一个完整但很小的 AI 算法：

```text
原始文本
  ↓
特征工程
  ↓
神经网络预测
  ↓
反向传播训练
  ↓
投票输出结果
```

它包含了很多 AI 算法的核心概念：

- 数据不能直接给模型，要先转成特征
- 模型通过权重表达判断规则
- 前向传播负责预测
- 反向传播负责学习
- 学习率控制更新幅度
- 多数投票可以提升整体稳定性

虽然这个神经网络很小，但它已经具备了机器学习程序的完整结构。
