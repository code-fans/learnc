# 洛谷 B3759 文本分类题解：用小神经网络判断英文和拼音

题目来源：洛谷 [`B3759 [信息与未来 2021] 文本分类`](https://www.luogu.com.cn/problem/B3759)

本文对应代码：`solution.cpp`。如果想提交更短的版本，可以参考 `aiEnglish.cpp`。

这道题要判断一组小写单词更像英文，还是更像汉语拼音。本文使用的方法是：先判断每个单词，再用多数投票得到整组答案。

## 题意

每组数据给出若干个只含小写字母的单词。

程序需要输出：

```text
English
```

或：

```text
Pinyin
```

例如：

```text
this problem has a simple solution
```

更像英文。

```text
zhe ge ti mu qi shi hen jian dan
```

更像拼音。

## 基本思路

一句话里有很多单词。单独看一个单词可能会判断错，但如果大多数单词都像英文，那么整句话通常就是英文。

所以流程是：

```text
读入一组单词
  ↓
逐个提取单词特征
  ↓
用神经网络判断每个单词
  ↓
统计英文票数
  ↓
英文票超过一半，输出 English，否则输出 Pinyin
```

核心代码：

```cpp
if (predict(word_feat(w)) > 0.5) eng++;
```

最后判断：

```cpp
cout << (eng > n / 2 ? "English" : "Pinyin") << '\n';
```

## 为什么可以这样判断

英文和拼音虽然都由小写字母组成，但字母组合习惯不同。

常见规律如下：

| 规律 | 倾向 |
|---|---|
| 有 `q`、`x` | 更像拼音 |
| 有 `zh`、`ch`、`sh` | 更像拼音 |
| 以 `s`、`d`、`k`、`t` 等结尾 | 更像英文 |
| 有 `th`、`br`、`st`、`ck` 等连续辅音 | 更像英文 |
| 拼音常以 `a/e/i/o/u/n/g/r` 结尾 | 可帮助判断 |

这些规律不是绝对正确，但合在一起就比较有用。

## 单词特征

神经网络不能直接处理字符串，所以要先把单词变成数字。

本题把每个单词变成 7 个特征：

| 编号 | 特征 | 说明 |
|---|---|---|
| `f[0]` | 词长 / 8 | 表示单词长度 |
| `f[1]` | 是否含 `q` | 有为 `1`，没有为 `0` |
| `f[2]` | 是否含 `x` | 有为 `1`，没有为 `0` |
| `f[3]` | 是否含 `zh/ch/sh` | 拼音中常见 |
| `f[4]` | 结尾是否不像拼音 | 英文中更常见 |
| `f[5]` | 是否有连续辅音 | 英文中更常见 |
| `f[6]` | 元音比例 | `a/e/i/o/u` 的占比 |

例如 `quick`：

```text
quick -> [0.625, 1, 0, 0, 1, 1, 0.4]
```

它含有 `q`，以 `k` 结尾，并且有 `ck` 连续辅音。

例如 `zhe`：

```text
zhe -> [0.375, 0, 0, 1, 0, 0, 0.5]
```

它含有 `zh`，这是明显的拼音特征。

## 小神经网络

程序使用一个很小的神经网络：

```text
7 个输入 -> 4 个隐藏节点 -> 1 个输出
```

也就是 `7-4-1` 结构。

输出是一个 `0` 到 `1` 之间的分数：

```text
接近 1：更像英文
接近 0：更像拼音
```

代码中用 `sigmoid` 函数把结果限制在 `0` 到 `1` 之间：

```cpp
double sigmoid(double x) {
    return 1.0 / (1.0 + exp(-x));
}
```

```text

调整参数

```

对应代码：

```cpp
for (int epoch = 0; epoch < 200; epoch++)

```

## 为什么要多数投票

有些单词本身很短，容易误判。

例如英文中的：

```text
in, to, he
```

也可能看起来像拼音。

拼音中的：

```text
men, de, he
```

也可能看起来像英文。

所以程序不让一个单词决定答案，而是让所有单词一起投票。这样更稳定。

## 正确性说明

本算法先把每个单词转换为 7 个能反映英文和拼音差异的特征。

神经网络根据这些特征给出一个分数。如果分数大于 `0.5`，就把该单词记为英文；否则记为拼音。

对整组单词，程序统计英文单词的数量。如果英文数量超过一半，说明大多数单词更像英文，输出 `English`；否则输出 `Pinyin`。

因此，算法的判断过程与题目要求一致。

## 复杂度分析

设一组数据中所有单词总长度为 `L`。

提取特征时，每个字符只处理一次，所以预测一组数据的时间复杂度是：

```text
O(L)
```

神经网络大小固定，训练样本和训练轮数也固定，所以训练部分可以看作常数时间。

额外空间复杂度是：

```text
O(1)
```
